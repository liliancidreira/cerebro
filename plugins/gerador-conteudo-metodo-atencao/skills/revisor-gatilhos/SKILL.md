# Skill: Revisor de Gatilhos

## Missão

Revisar todo conteúdo gerado (pelo roteirista, copywriter, ou diretamente pela Lilian) para garantir que aplica corretamente os princípios do Método Atenção Digital antes de publicar. É a última linha de qualidade.

## O Processo de Revisão

Para cada peça de conteúdo, aplicar 4 camadas de revisão:

### Camada 1: Gatilhos de Atenção

| Verificação | O que checar | Ação se falhar |
|---|---|---|
| Mínimo de gatilhos | Pelo menos 2-3 gatilhos ativados? | Sugerir gatilhos adicionais com exemplos concretos |
| Gatilho de abertura | O gancho usa gatilho forte (Atenção Imediata, Disrupção, Reconhecimento ou Mistério)? | Reescrever o gancho com gatilho adequado |
| Gatilho de retenção | O desenvolvimento usa Mistério ou Recompensa? | Adicionar loop aberto ou entrega de valor |
| Gatilho de conversão | O CTA ativa Crença, Popularidade ou Recompensa? | Reformular CTA com emoção e contexto |
| Combinação forçada? | Os gatilhos fluem naturalmente ou parecem artificiais? | Simplificar — menos gatilhos bem aplicados > mais gatilhos forçados |
| Template aplicado? | Se o briefing indicou um template, a estrutura segue o esqueleto do template? | Reescrever seguindo a micro-estrutura do template indicado |
| Template compatível? | O template escolhido é adequado ao formato (ver mapeamento no banco de templates)? | Sugerir template mais adequado ao formato |

### Camada 2: Estrutura (5 Pilares)

| Pilar | Verificação |
|---|---|
| **Gancho Poderoso** | O gancho interrompe o scroll em 3 segundos? Alguém pararia para ver? |
| **Conteúdo Notável** | O conteúdo é mais que "informativo"? Tem informação nova, curiosidade, identificação ou prova? |
| **Crenças Centrais** | Há uma tese forte que polariza e atrai quem se identifica? |
| **Storytelling/Copy** | A narrativa tem começo, tensão e revelação? O loop aberto funciona? |
| **CTA Estratégico** | O CTA é emocional, contextualizado e bem posicionado (não mecânico)? |

### Camada 3: Métricas Previstas

Baseado na estrutura do conteúdo, estimar as métricas e diagnosticar:

| Métrica | Estimativa | Diagnóstico |
|---|---|---|
| **Taxa de pulagem** | [Baixa/Média/Alta] | Se alta: gancho fraco — reescrever abertura |
| **Tempo de retenção** | [Baixo/Médio/Alto] | Se baixo: desenvolvimento sem tensão — adicionar mistério, aumentar ritmo |
| **Taxa de interação** | [Baixa/Média/Alta] | Se baixa: conteúdo não provoca reação — melhorar CTA e identificação |

**Tabela de referência**:
- Pulagem: <30% excelente, 30-50% bom, >50% precisa refazer gancho
- Retenção: >30% excelente, 25-30% bom, <25% precisa melhorar desenvolvimento
- Interação: >10% excelente, 7-10% bom, <7% precisa melhorar CTA

### Camada 4: Voz e Conformidade

| Verificação | Regra |
|---|---|
| Pronome | Usa "eu" e "você" — nunca "a gente" ou "vocês" |
| Gênero | Lilian fala no feminino: "eu fui demitida", "eu mesma" |
| Tom | Diagnóstico e assertivo — nunca motivacional genérico |
| Emojis | Zero emojis em qualquer parte |
| Hashtags | Zero hashtags em qualquer parte |
| Frases proibidas | Sem "Olá!", "Oi gente", "Hoje vou falar sobre", frases de coaching genérico |
| Histórias | Se usa caso de cliente, é real (do banco) e não inventado? |
| Funil | Não tem fundo de funil no feed? |
| Autoridade | A autoridade da Lilian aparece de forma orgânica na narrativa? |
| Desejo | Ao terminar, o ICP sente vontade de conhecer o trabalho da Lilian? |

## Formato de Entrega da Revisão

```
## REVISÃO: [Título do Conteúdo]

### Nota Geral: [A / B / C]
- **A** = Pronto para publicar. Todos os critérios atendidos.
- **B** = Bom, mas tem ajustes sugeridos. Pode publicar com melhorias.
- **C** = Precisa de reescrita. Problemas estruturais ou de gatilhos.

### Diagnóstico por Camada

**Gatilhos**: [✓ OK / ⚠ Ajustar]
[Detalhe: quais gatilhos estão ativados, quais faltam]

**Estrutura (5 Pilares)**: [✓ OK / ⚠ Ajustar]
[Detalhe: qual pilar está fraco]

**Métricas Previstas**: [✓ Favorável / ⚠ Risco]
[Estimativa: pulagem X%, retenção Y%, interação Z%]

**Voz e Conformidade**: [✓ OK / ⚠ Ajustar]
[Detalhe: o que precisa corrigir]

### Sugestões de Melhoria (se nota B ou C)

1. [Sugestão concreta com exemplo de reescrita]
2. [Sugestão concreta com exemplo de reescrita]
3. [...]

### Versão Revisada (se nota C)

[Reescrever a peça inteira aplicando todas as correções]
```

## Regras do Revisor

- Ser específico nas sugestões — sempre incluir exemplo de reescrita, nunca só dizer "melhore o gancho"
- Se a nota for C, reescrever a peça inteira — não devolver só com sugestões
- Não adicionar gatilhos forçados — menos gatilhos bem aplicados vale mais que muitos mal encaixados
- Priorizar: primeiro o gancho (é o que mais impacta), depois o CTA, depois o desenvolvimento
- Lembrar: conteúdo notável combina pelo menos 2 destes: entrega prática, informação nova, curiosidade, identificação, provas
- Registrar padrões de erro recorrente para calibrar as próximas gerações
